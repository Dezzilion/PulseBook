# Архітектура системи PulseBook

## 3.1 Загальна архітектура системи

Система PulseBook побудована за принципами мікросервісної архітектури. Це дозволяє забезпечити високу масштабованість, незалежність компонентів та зручність підтримки.

Основні шари системи:

- Frontend — клієнтська частина (веб-інтерфейс)
- Backend — серверна частина (набір мікросервісів)
- Data Layer — база даних та сховища
- AI Layer — модулі штучного інтелекту
- Integration Layer — інтеграції з зовнішніми сервісами

Для розгортання використовується хмарна інфраструктура, що дозволяє автоматично масштабувати систему під час пікових навантажень.

## 3.2 Використані технології

### Frontend

- Next.js 15 (App Router) + TypeScript
- Tailwind CSS + Shadcn/ui
- Zustand (стан менеджмент)
- TanStack Query (для роботи з API)
- Mermaid (для візуалізації діаграм)

### Backend

- NestJS (Node.js) — основний фреймворк
- TypeScript
- PostgreSQL + Prisma ORM
- Redis (для кешування та черг)
- JWT + Refresh tokens (автентифікація)

### Штучний інтелект

- Python + FastAPI (окремий мікросервіс рекомендацій)
- Scikit-learn + TensorFlow / PyTorch
- Hybrid recommendation system (collaborative + content-based + deep learning)
- Explainable AI (SHAP / LIME для пояснення рекомендацій)

### Інфраструктура

- Docker + Docker Compose
- Kubernetes (для продакшену)
- Vercel (frontend) + Railway / Render (backend)
- PostgreSQL + Redis Cloud

## 3.3 Реалізовані функціональні модулі

- Модуль автентифікації та профілю користувача
- Модуль управління подіями
- Рекомендаційна система (основний ШІ-модуль)
- Модуль бронювання місць
- Модуль оплати та генерації квитків
- Модуль аналітики

## 3.4 Особливості реалізації

- Адаптивність до нестабільного інтернету — progressive enhancement та часткова offline-робота (PWA)
- Інклюзивність — підтримка WCAG 2.2
- Захист від ботів — rate limiting, behavioral analysis, прості нейромережеві моделі
- Масштабованість — Redis для кешування та черг, горизонтальне масштабування backend-сервісів

## 3.5 Поточний стан реалізації

- Повноцінний frontend-інтерфейс
- Backend з основними мікросервісами
- База даних та ORM
- Базова версія рекомендаційної системи
- Модуль бронювання та оплати (тестовий режим)
- Інтерактивні схеми залів

### У розробці

- Розширена аналітика для організаторів
- Глибша інтеграція з сервісом Дія
